# CampusFlow

## Attendance Permission Management System

CampusFlow is a frontend-based attendance permission management system. It simplifies the process of submitting, reviewing, and managing student attendance requests.

The application offers different dashboards and workflows for students, employees, HODs, and administrators. It shows how a real college permission management platform can function.

---

## About The Project

CampusFlow was developed during a **3-Day DTAI Workshop**.

This project serves as a **demo/prototype version** to highlight frontend development skills, UI design, and workflow implementation.

**Note:**
- This application uses sample and duplicate data for demonstration purposes.
- No real student information or college database is connected.
- Authentication, requests, and approvals are simulated on the frontend.
- The project emphasizes user experience and workflow demonstration.

---

## Features

### Student Module
- Student login interface
- Submit attendance permission requests
- Add event details and descriptions
- Attach supporting documents
- Track request status

### Employee Module
- View approved requests
- Filter requests by sections
- Monitor student permissions

### HOD Module
- View student requests
- Review request details
- Approve or reject requests
- Filter requests by sections

### Admin Module
- Manage application-level data
- View system information
- Demonstrate administrative workflow

## Demo Limitations

Since this project was developed during a 3-day workshop:

- No backend integration
- No real database
- No cloud storage
- No real authentication system
- Data is stored temporarily in JavaScript state
- Demo accounts and requests are pre-configured
